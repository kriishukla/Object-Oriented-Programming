IHCI Monsoon 2022 - PROJECT GUIDELINES

The course project constitutes 25 marks of your final score(out of 100 marks).

Project Proposal

Submission Date - 7th December 

Marks assigned: 3 marks

What is expected:

  • Problem Statement [1 mark]
  • Background of the problem and motivation [1 mark]
  • Aspects that the solution will cover/ the scope of the project [1 mark]

The project is further then divided into 4 phases.

First Project Submission:

Submission Date - 21st December

Marks assigned: 4  marks

The aim of this submission is to understand and implement the first two phases of the Double Diamond Model, i.e., Discover and Define.

What is required:

  • Requirement Gathering [2 marks] – Identifying stakeholders and their roles, challenges, requirements and expectations of users through tools like interviews, mindmaps, use-case diagrams etc.
  • Personas and Scenarios [1 mark] – developing different user archetypes to understand and empathise with users through business and personal contexts
  • Problem Understanding[1 mark] – Elaborate on the proposed solution, its target users, and the specific requirements it would fulfil

Second Project Submission: Submission Date - 28th December

Marks assigned: 4  marks

The aim of this submission is to understand and implement the first three phases of the Double Diamond Model, i.e., Discover, Define and Develop.

  • Refined [Requirement gathering, Persona, Problem Understanding] [1 mark]
  • Low-Fidelity Design with some testing [2 marks] – Hand-drawn sketches or simple wireframes on Figma to present the features and features and architecture of the application. Creating task flows with some amount of testing of the lo-fi design
  • Alternate Design [1 mark] – Creating alternate design(s) and task flows based on the feedback and testing

Third Project Submission: Submission Date - 11th January

Marks assigned: 5  marks

The aim of this submission is to understand and implement the first four phases of the Double Diamond Model, i.e., Discover, Define, Develop and Deliver.

  • Refined [Requirement gathering, Persona, Problem Understanding, Low Fi Design] [1 marks]
  • High-Fidelity Design with some evaluation [4 marks]– creating functional and interactive prototypes on Figma and evaluating the product by user testing it along with documenting the user evaluation.

Final Project Submission: Submission Date - 18th January

Marks assigned: 9  marks

  • Refined [Requirement gathering, Persona, Problem Understanding, Low Fi Design, Hi-Fi Design, User Testing and Evaluation]

Refine all the aspects you have covered in the previous submissions based on the user feedback and evaluation and document it. Present the entire project with the final solution/application and its design process.

NOTE:

  • All submissions should be submitted on Google Docs.
  • Document your submission in font size 11 with 1.5 spacing
  • Submissions for all the phases should include the documentation, the data, the design etc.
  • The same person from each group should submit the submissions every time for no confusion.
  • As per the given timeline, it’s suggested that groups start timing and planning the work accordingly in advance.
